# Anti-Pattern Registry

## Visual Anti-Patterns
- Purple-to-blue gradient on white/light backgrounds
- 3-column card grids: icon → heading → body text
- Glassmorphism / frosted glass used decoratively
- Geometric abstract hero illustrations (the "SaaS blob" aesthetic)
- Bento grid dashboard layouts (Vercel/Linear tribute acts)
- Excessive blur, glow, or shadow for "depth"
- Inter + system sans-serif with zero typographic identity
- Stock photography or AI-generated "diverse team" imagery
- Floating mockup devices at angles
- Gradient mesh backgrounds
- The "dark mode with neon accents" startup aesthetic
- Rounded-xl everything with ring-offset hover states (Tailwind default)

## Copy Anti-Patterns
- "Unlock the power of..."
- "Seamlessly integrate..."
- "Take your ___ to the next level"
- "Whether you're a [A] or [B]..."
- "Built for teams who..."
- "The all-in-one platform for..."
- "Supercharge / Revolutionize / Empower / Elevate"
- "In today's fast-paced world..."
- Starting with "Imagine..." or "What if..."
- Three-bullet CTA sections
- "Trusted by X+ teams worldwide" with grayscale logos
- The word "leverage" used unironically
- "We believe that..." mission statements that say nothing

## Structural Anti-Patterns
- Hero → Features (3 cards) → Testimonials → CTA → Footer (SaaS skeleton)
- Pricing table with "Most Popular" badge on middle tier
- FAQ accordion at bottom as afterthought
- "How it works" in three numbered steps with icons
- Hamburger menu on desktop
- Cookie-cutter 404 pages with cartoon illustrations
- Login/signup forms identical to every OAuth starter

## Code Anti-Patterns (Taste-Specific)
- File structure mirroring create-next-app / create-react-app defaults
- Component names so generic they could be in any project (Card, Hero, Section)
- CSS class names describing appearance not purpose (.blue-button, .large-text)
- Config files copy-pasted from docs without project-specific tuning
- README.md that reads like a framework tutorial, not a project document
